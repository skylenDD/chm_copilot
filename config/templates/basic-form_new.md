# 基础表单页面模板

## 模板信息
- **ID**: basic-form
- **分类**: form  
- **版本**: 1.0.0
- **适用场景**: 用户注册、登录、信息编辑等基础表单场景

## 使用的组件

### 核心组件结构
- **根容器 (root)**: 页面根组件，负责整体布局和主题设置
- **表单容器 (fh_form)**: 包裹所有表单字段，处理表单提交逻辑
- **文本输入框 (fh_input-text)**: 用于单行文本输入（如姓名、年龄）
- **下拉框 (fh_select)**: 用于选项选择（如性别、婚否状态）
- **文本域 (textarea)**: 用于多行文本输入（如简介、描述）
- **按钮 (fh_button)**: 用于表单操作（提交、重置）

## 页面结构描述

### 整体布局
- 页面采用垂直布局，表单居中显示
- 表单容器包含多个字段行，每行可包含1-2个字段
- 底部包含操作按钮区域（重置、提交按钮）

### 字段组织
- **单行字段**: 姓名、年龄等使用单独的文本输入框
- **双列布局**: 性别和婚否状态并排显示，节省空间
- **多行字段**: 简介使用文本域组件，支持多行输入
- **操作区域**: 按钮右对齐，符合用户操作习惯

## 事件调用说明

### 预定义事件处理函数
- **handleSubmit(ctx)**: 处理表单提交事件，由提交按钮触发
  - 参数: `ctx` (组件上下文信息)
  - 功能: 收集表单数据并执行提交逻辑
  
- **handleReset(ctx)**: 处理表单重置事件，由重置按钮触发  
  - 参数: `ctx` (组件上下文信息)
  - 功能: 重置所有表单字段为初始值

### 组件事件绑定
- **文本输入框**: 
  - `onChange` 事件绑定到字段值变化处理
  - `onFocus` 事件可用于字段聚焦时的特殊处理
  
- **下拉框**: 
  - `onChange` 事件绑定到选项选择处理
  
- **按钮**: 
  - `onClick` 事件分别绑定到 `handleSubmit` 和 `handleReset` 函数

## 数据源配置
- **formData**: 主要表单数据变量，用于存储和绑定所有表单字段值
- **页面参数**: 包含页面基本信息（title, theme等）
- **URL参数**: 可用于接收外部传入的初始化数据

## 事件函数生成要求
根据组件事件生成规范，必须在 `actions.module.source` 中生成以下导出函数：
- `export function handleSubmit(ctx) { console.log('handleSubmit', ctx); }`
- `export function handleReset(ctx) { console.log('handleReset', ctx); }`

所有函数参数签名必须严格匹配组件配置文件中的实际定义。
